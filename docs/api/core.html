lux.core.app.execute_from_config(config_file, argv=None, **params)[source]¶

Create and run an App from a config_file.

This is the function to use when creating the script which runs your web applications:

import lux

if __name__ == '__main__':
    lux.execute_from_config('path.to.config')
Parameters:config_file – the python dotted path to the config file for setting up a new App. The config file should be located in the python module which implements the main application of the web site.

Parameter¶

class lux.core.extension.Parameter(name, default, doc)[source]¶

Class for defining a lux parameter within a lux Extension.

Parameters are specified when creating an Extension in a declarative style (as class attributes of the extension). For example:

from lux import Extension, Parameter

class MyExtension(Extension):
    title = Parameter('Hello', 'The title to use in the home page')
Parameters:
  • default – the default value of the parameter. This is the value used by the framework when the parameter is not found in the config file.
  • doc – a documentation string for the parameter.

Parameters are case insensitive.

The Template defines a family of classes which can be used to build HTML elements in a pythonic fashion. No template specific language is required, instead a template for an html element is created by adding children Template to a parent one.

Template¶

class lux.core.templates.Template(*children, **parameters)[source]¶

A factory of Html objects.

>>> simple = Template(Context('foo', tag='span'), tag='div')
>>> html = simple(cn='test', context={'foo': 'bla'})
>>> html.render()
<div class='test'><span data-context='foo'>bla</span></div>
tag¶

An optional HTML tag for the outer element of this template. If not specified, this template is a container of other templates and no outer element is rendered.

key¶

An optional string which identify this Template within other templates. It is also used for extracting content from the context dictionary passed to the template callable method.

children¶

List of Template objects which are rendered as children of this Template

parameters¶

An attribute dictionary containing all key-valued parameters passed during initialisation. These parameters are used when building an Html element via the callable method.

it is initialised by the init_parameters() method at the end of initialisation.

init_parameters(tag=None, classes=None, **parameters)[source]¶

Called at the and of initialisation.

It fills the parameters attribute. It can be overwritten to customise behaviour.

keys()[source]¶

Generator of keys in this Template

get(key)[source]¶

Retrieve a children Template with Template.key equal to key.

The search is done recursively and the first match is returned. If not available return None.

Context¶

class lux.core.templates.Context(key, *children, **params)[source]¶

A Template which enforces the key attribute.

Fore example:

>>> from lux import Context
>>> template = Context('foo', tag='div')
>>> template.key
'foo'
>>> html = template(context={'foo': 'pippo'})
>>> html.render()
<div>pippo</div>

    ...)

class lux.core.templates.RowTemplate(*children, **params)[source]¶

A RowTemplate is a container of ColumnTemplate elements. It should be placed inside a GridTemplate for better rendering.

Parameters:column – Optional parameter which set the column attribute.
column¶

It can be either 12 or 24 and it indicates the number of column spans available.

Default: 12

Column Template¶

class lux.core.templates.ColumnTemplate(*children, **params)[source]¶

A template to place inside a RowTemplate.

Parameters:
  • span=1 – fraction indicating how much this template extend across its row container. For example 0.5 is half span. Must be between 0 and 1.
  • device=’md’ –

    The device which render the column with the correct span. Valid options are

    • xs extra small (phones)
    • sm small (tablets)
    • md medium (desktops)
    • lg large (desktops)
